# AI-SDLC Framework v3 — Gemini CLI Instructions

> **Version:** 3.3
> **Last Updated:** 2026-06-09
> **Audience:** Gemini CLI operating inside a project that uses this framework
> **See Also:** [docs/framework-reference.md](docs/framework-reference.md), [CLAUDE.md](CLAUDE.md), [AGENTS.md](AGENTS.md)

## First Action — ALWAYS

Read `ai-sdlc/tracking/project-state.md` before doing anything.

## Commands

All commands are slash commands backed by `ai-sdlc/commands/ai-*.md`. Gemini reads the command's frontmatter (`reads:`, `updates:`, `requires_approval:`, `token_budget:`, `phase_constraint:`) to decide what to load.

Type `/ai-help` for the full list (37 slash commands — see `docs/framework-reference.md`).

## How to Work

1. Read `ai-sdlc/tracking/project-state.md`.
2. Read `ai-sdlc/playbooks/_index.md`.
3. Read the relevant phase file under `ai-sdlc/phases/`.
4. Execute the appropriate command.
5. Update `ai-sdlc/tracking/project-state.md` after any state change.
6. Run `/ai-sync` to regenerate derived files.

## File Paths

All framework files live under `ai-sdlc/`. References like `tracking/project-state.md` resolve to `ai-sdlc/tracking/project-state.md` — prepend the prefix.

## Tracking Rules

**Source-of-truth** (commands write here): `tracking/project-state.md`, `tracking/story-index.md`, `tracking/changelog.md`, `tracking/decision-log.md`, `tracking/risks.md`, `tracking/anti-patterns.md`.

**Derived** (regenerated by `/ai-sync` — never write directly): `tracking/dashboard-data.json`, `tracking/sprint-backlog.md`, `tracking/rtm.md`.

**Session:** `tracking/recovery.md`.

## Token Efficiency

1. Frontmatter-first reading.
2. Follow `token_budget` (small -> frontmatter only, medium -> selective, large -> full reads).
3. Conditional reads — only the files needed for the selected mode/flag.
4. Cache shared context once per multi-step command.
5. Never write derived files directly.

## Approval Rules

See `ai-sdlc/config/workflow-rules.yaml`. Default: read-only + in-sprint code = autonomous; adding stories, architecture changes, gate passage, phase transitions = human approval.

## Session Recovery

On interrupted restart: read `ai-sdlc/tracking/recovery.md` -> read `project-state.md` -> resume from checkpoint -> clear recovery when done.

## Evidence Packages

Quality gates verify canonical evidence package completeness before phase transitions:

- **Discover-to-B0**: charter, requirements, architecture-package, observability-intent, b0-readiness-checklist
- **Ship S1–S3**: s1-validation-evidence, s2-hardening-evidence, s3-deploy-readiness
- **Evolve**: feedback-log, iteration-plan, tech-debt-register, performance-review

ANY-phase commands (`ai-status`, `ai-help`, `ai-risk`, `ai-change`, `ai-handoff`) read this package readiness to provide informed guidance.

## Generated Skills

Run `/ai-scaffold skills` to create project-specific pattern overlays in `ai-sdlc/playbooks/generated/`, including the four observability skills.

- Most generated overlays are strongest after B0.
- Observability skills should start from Discover intent and then be refreshed after B0 so the generated constraints reflect proven evidence.

These are consulted by `/ai-build`, `/ai-preflight`, `/ai-review`, `/ai-secure`, `/ai-test`, `/ai-wire`, `/ai-harden`, `/ai-monitor`, `/ai-deploy`, `/ai-debt`, `/ai-story`, and `/ai-ui`.

## Gemini-Specific Tool Mapping

- `read_file` -> read framework files
- `write_file` -> update tracking and source files
- `run_terminal_command` -> execute build/test commands
- `activate_skill` -> load skills or playbooks
- `search_files` -> find content across codebase
