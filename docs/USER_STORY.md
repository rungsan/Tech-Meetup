## Module 1: Authentication / Login

### US-001: เข้าสู่ระบบด้วย Azure AD

**As a** ABC Staff (Car Ins Staff / UW / Admin),  
**I want to** เข้าสู่ระบบตรวจสภาพรถยนต์ผ่าน Microsoft Azure Active Directory,  
**So that** ฉันสามารถใช้งานระบบได้อย่างปลอดภัยโดยใช้บัญชีองค์กรที่มีอยู่แล้ว

**Acceptance Criteria:**
- [ ] หน้าจอ Login แสดงปุ่ม "Login" เพื่อนำไปยัง Microsoft Azure AD
- [ ] เมื่อกด Login ระบบแสดงหน้าจอ Microsoft Azure AD สำหรับยืนยันตัวตน
- [ ] หลังยืนยันตัวตนสำเร็จ ระบบตรวจสอบ Role จาก ADD Group
- [ ] ผู้ใช้ที่มีสิทธิถูกต้องเข้าสู่หน้า Dashboard ได้ทันที
- [ ] ผู้ใช้ที่ไม่มีสิทธิระบบแสดงข้อความแจ้งเตือน "ไม่สามารถเข้าสู่ระบบได้"
- [ ] ผู้ใช้ที่อยู่ใน ADD Group มากกว่า 1 กลุ่ม จะไม่ได้รับสิทธิเข้าใช้ระบบ

## Module 2: Dashboard

### US-002: ดู Dashboard สรุปสถานะงาน

**As a** ABC Staff,  
**I want to** ดู Dashboard ที่แสดงสรุปสถานะงานตรวจสภาพรถทั้งหมด,  
**So that** ฉันสามารถทราบภาพรวมของงานและวางแผนการทำงานได้อย่างมีประสิทธิภาพ

**Acceptance Criteria:**
- [ ] Dashboard แสดง Card จำนวนงานแต่ละสถานะ: งานเข้าใหม่, ติดตามนัดหมายลูกค้า, ส่ง SV ออกตรวจสอบ, ติดตาม SV, รอผลตรวจรถยนต์, ตรวจรถยนต์แล้ว, ไม่ส่ง SV, ยกเลิก
- [ ] เมื่อกด Card แต่ละสถานะ ระบบนำไปยังหน้า "รายการตรวจสภาพ" พร้อม Filter สถานะนั้นๆ อัตโนมัติ
- [ ] แสดงกราฟแท่งปริมาณการตรวจสภาพรถเปรียบเทียบ 3 ปีย้อนหลัง (รวมปีปัจจุบัน)
- [ ] แสดงกราฟแท่งจำนวนค่าใช้จ่ายเปรียบเทียบ 3 ปีย้อนหลัง
- [ ] แสดงตัวอย่างรายการของฉัน 5 รายการล่าสุด พร้อม Link "ไปหน้ารายการ"
- [ ] แสดงตัวอย่างรายการตรวจสภาพ 5 รายการล่าสุด พร้อม Link "ไปหน้ารายการ"

### US-003: กรอง Dashboard ตามช่วงเวลา

**As a** ABC Staff,  
**I want to** กรอง Dashboard ตามช่วงเวลาที่ต้องการ,  
**So that** ฉันสามารถวิเคราะห์ข้อมูลในช่วงเวลาที่สนใจได้

**Acceptance Criteria:**
- [ ] มีตัวกรองช่วงเวลา: กำหนดเอง, 30 วันย้อนหลัง, 60 วันย้อนหลัง, 90 วันย้อนหลัง
- [ ] รองรับการระบุวันที่เริ่มต้น-สิ้นสุดแบบ Custom
- [ ] กดปุ่ม "ตกลง" เพื่อแสดงข้อมูลตามช่วงเวลาที่เลือก
- [ ] กดปุ่ม "ล้างค่า" เพื่อรีเซ็ตกลับค่า Default
- [ ] Dashboard ส่วน Summary แสดง: ปริมาณการแจ้งงานแต่ละฝ่ายธุรกิจ, Top 5 Survey (Pie Chart), สถานะงานทั้งหมด (Bar Chart)

## Module 3: แจ้งตรวจสภาพรถใหม่

### US-004: สร้างรายการตรวจสภาพรถคันเดียว (บุคคลทั่วไป)

**As a** ABC Staff,  
**I want to** สร้างรายการแจ้งตรวจสภาพรถยนต์ใหม่สำหรับลูกค้าบุคคลทั่วไป 1 คัน,  
**So that** ระบบสามารถส่งงานให้ Surveyor ดำเนินการตรวจสภาพรถได้

**Acceptance Criteria:**
- [ ] ต้องระบุ Source (บังคับ), ฝ่ายธุรกิจ (บังคับ), Agent (ไม่บังคับ) รองรับ Auto Complete
- [ ] สามารถเลือก Checkbox "แจ้งงานกลุ่ม Fleet" เพื่อเปลี่ยนเป็นโหมด Fleet
- [ ] ข้อมูลลูกค้า: ประเภท (บุคคลทั่วไป/นิติบุคคล), ชื่อ-นามสกุล/ชื่อนิติบุคคล, มือถือลูกค้า
- [ ] ข้อมูลรถ: ทะเบียน (ห้ามใส่ขีด/เว้นวรรค), จังหวัด, ยี่ห้อ, รุ่น, เลขตัวถัง, ประเภท (Non EV/EV)
- [ ] วันที่เริ่มต้นคุ้มครอง (บังคับ)
- [ ] สถานะการนัด (บังคับ): ยังไม่ได้นัดลูกค้า, นัดลูกค้า
- [ ] หมายเหตุ ไม่ส่ง Survey: มีตัวเลือก (ลูกค้าส่งรูป, ตัวแทน-นายหน้าส่งรูป, สาขาตรวจสภาพรถเอง, ฯลฯ)
- [ ] อีเมลผู้แจ้งงาน: รองรับหลายอีเมลโดยใช้ ";" คั่น
- [ ] กดปุ่ม "บันทึก" ได้เมื่อกรอกข้อมูลบังคับครบถ้วน
- [ ] ระบบแจ้งเตือนยืนยันก่อนบันทึก และแจ้งเตือนเมื่อบันทึกสำเร็จ/ไม่สำเร็จ
- [ ] รายการที่บันทึกสำเร็จแสดงที่หน้ารายการตรวจสภาพ

### US-005: สร้างรายการตรวจสภาพรถป้ายแดง

**As a** ABC Staff,  
**I want to** สร้างรายการตรวจสภาพสำหรับรถป้ายแดง,  
**So that** ระบบสามารถจัดการกรณีรถที่ยังไม่ได้จดทะเบียนได้ถูกต้อง

**Acceptance Criteria:**
- [ ] มี Checkbox "รถป้ายแดง" ในส่วนข้อมูลรถยนต์
- [ ] เมื่อเลือกรถป้ายแดง ระบบแสดงจังหวัดเป็น "99" และไม่สามารถแก้ไขได้
- [ ] ฟิลด์อื่นๆ ยังกรอกได้ตามปกติ

### US-006: สร้างรายการตรวจสภาพแบบ Fleet

**As a** ABC Staff,  
**I want to** สร้างรายการตรวจสภาพรถยนต์หลายคัน (Fleet) ในรายการเดียว,  
**So that** ฉันสามารถจัดการงานตรวจสภาพสำหรับลูกค้าองค์กรที่มีรถหลายคันได้อย่างมีประสิทธิภาพ

**Acceptance Criteria:**
- [ ] เลือก Checkbox "แจ้งงานกลุ่ม Fleet" เพื่อเปิดใช้งานโหมด Fleet
- [ ] ระบุจำนวนรถที่ต้องการสร้าง (2-10 คัน) ด้วยปุ่ม +/- หรือพิมพ์ตัวเลข
- [ ] กดปุ่ม "สร้าง" ระบบแสดง Field ข้อมูลรถยนต์ตามจำนวนที่ระบุ
- [ ] สามารถกดปุ่ม "สร้าง" ได้มากกว่า 1 ครั้ง ระบบจะนับต่อจากจำนวนเดิม
- [ ] สามารถกดปุ่ม "ลบ" เพื่อลบข้อมูลรถแต่ละคัน
- [ ] กดปุ่ม "บันทึก" เมื่อกรอกข้อมูลรถทุกคันครบถ้วน
- [ ] ระบบแจ้งเตือนหากมีเลขทะเบียนซ้ำหรือไม่ถูกต้อง
- [ ] รายการ Fleet ที่บันทึกสำเร็จแสดงที่หน้ารายการตรวจสภาพ

## Module 4: รายการของฉัน (My Work)

### US-007: ดูรายการงานที่รับผิดชอบ

**As a** ABC Staff,  
**I want to** ดูรายการตรวจสภาพรถที่ฉันรับผิดชอบดูแล,  
**So that** ฉันสามารถติดตามและจัดการงานของตัวเองได้อย่างมีระบบ

**Acceptance Criteria:**
- [ ] แสดงรายการงานที่ฉัน "Assign to me" เท่านั้น
- [ ] สามารถค้นหาตาม: วันที่, Job ID, ประเภทลูกค้า, ชื่อ/นามสกุล, ทะเบียนรถ, เลขตัวถัง, Source, Agent, สถานะงาน
- [ ] แสดงคอลัมน์: No., Job ID, วันที่บันทึก, วันที่นัดหมาย, ทะเบียน, วันคุ้มครอง, ชื่อลูกค้า, Source, Agent, สถานะ, หมายเหตุ
- [ ] มี icon แสดงเมื่อมีข้อความใหม่
- [ ] สามารถ Export รายการเป็น Excel ได้
- [ ] รองรับ Pagination (10, 20, 50, 100 รายการ/หน้า)

### US-008: Assign งานให้ตนเอง

**As a** ABC Staff,  
**I want to** รับมอบหมายงานตรวจสภาพรถให้ตัวเองจากหน้ารายการตรวจสภาพ,  
**So that** งานนั้นจะปรากฏในหน้า "รายการของฉัน" และฉันสามารถติดตามได้

**Acceptance Criteria:**
- [ ] กดปุ่ม "Assign to me" ที่รายการตรวจสภาพ
- [ ] ระบบแสดง Modal ยืนยันการมอบหมายงาน
- [ ] กด "ยืนยัน" งานปรากฏที่หน้า "รายการของฉัน" ทันที
- [ ] กด "ยกเลิก" ระบบไม่มอบหมายงานและกลับสู่หน้ารายการ

### US-009: ดำเนินการขั้นตอน "ติดตามนัดหมายลูกค้า"

**As a** ABC Staff,  
**I want to** บันทึกข้อมูลการนัดหมายลูกค้าเพื่อตรวจสภาพรถ,  
**So that** ระบบสามารถติดตามวันนัดหมายและเปลี่ยนสถานะงานได้ถูกต้อง

**Acceptance Criteria:**
- [ ] เมื่อเปิด Job สถานะ "ติดตามนัดหมายลูกค้า" แสดงแถบเมนู: รายละเอียด, Job History, ข้อความ, Car Inspect Image (กรณีที่เกี่ยวข้อง)
- [ ] แถบ "รายละเอียด" แสดงข้อมูลงาน Job Information, ข้อมูลลูกค้า, ข้อมูลรถยนต์
- [ ] สามารถระบุข้อมูลนัดหมาย: วันที่นัด, เวลา, สถานที่, หมายเหตุ
- [ ] กรณีงาน Fleet สามารถเลือก Job ID ย่อยจาก Dropdown
- [ ] กดปุ่ม "Complete" ระบบแสดง Modal ยืนยัน ก่อนเปลี่ยนสถานะ
- [ ] สถานะเปลี่ยนเป็น "ส่ง SV ออกตรวจสอบ" เมื่อกด "ยืนยัน"
- [ ] สามารถเลือกสถานะ "ยกเลิก" ระบบเก็บประวัติและแสดง Job ID สถานะ "ยกเลิก"

### US-010: ส่ง SV ออกตรวจสอบ

**As a** ABC Staff,  
**I want to** ส่งงานตรวจสภาพรถให้บริษัท Surveyor ดำเนินการ,  
**So that** Surveyor ได้รับงานและสามารถดำเนินการตรวจสภาพรถได้

**Acceptance Criteria:**
- [ ] แสดงข้อมูล Survey ที่ได้รับมอบหมาย
- [ ] สามารถดูและแก้ไขราคาตรวจสภาพรถ
- [ ] สามารถส่งข้อความ SMS ให้ลูกค้าได้
- [ ] แสดง Tab "Job History" บันทึกการดำเนินงานทั้งหมด
- [ ] กดปุ่ม "Complete" ระบบส่งงานไปยัง E-Motor Survey อัตโนมัติ
- [ ] ระบบแจ้งเตือนเมื่อส่งงานสำเร็จ

### US-011: ติดตาม SV

**As a** ABC Staff,  
**I want to** ติดตามสถานะของ Surveyor ที่รับงานไปดำเนินการ,  
**So that** ฉันทราบว่างานอยู่ในสถานะใดและ Surveyor ได้รับงานหรือไม่

**Acceptance Criteria:**
- [ ] แสดงสถานะการรับงานของ Surveyor
- [ ] แสดงข้อมูล Job ครบถ้วน
- [ ] กดปุ่ม "Complete" เพื่อเปลี่ยนสถานะเป็น "รอผลตรวจรถยนต์"

### US-012: รอผลตรวจสภาพและตรวจสอบรูปภาพ

**As a** ABC Staff,  
**I want to** ตรวจสอบรูปผลตรวจสภาพรถที่ Surveyor ส่งมา และอนุมัติหรือแจ้งให้แก้ไข,  
**So that** ผลการตรวจสภาพมีความถูกต้องสมบูรณ์ก่อนปิดงาน

**Acceptance Criteria:**
- [ ] แสดงแถบ "Car Inspection Image" เพื่อดูรูปตรวจสภาพ
- [ ] สามารถ Preview รูปขนาดใหญ่
- [ ] สามารถกด "Revise" เพื่อระบุ Comment ให้ Surveyor แก้ไขรูป
- [ ] เมื่อ Surveyor แก้ไขแล้ว ระบบแสดงรูปก่อนแก้ไขและรูปปัจจุบันเปรียบเทียบกัน
- [ ] สามารถเลือก "รายงานผล" เพื่อบันทึกผลการตรวจสภาพ
- [ ] กดปุ่ม "Complete" ระบบเปลี่ยนสถานะเป็น "ตรวจรถยนต์แล้ว"

### US-013: จัดการกรณีลบรอย Remark (ส่งกลับนัดหมายใหม่)

**As a** ABC Staff,  
**I want to** ดำเนินการกรณีที่ต้องลบรอย Remark และส่งกลับให้ลูกค้านัดหมายใหม่,  
**So that** รถยนต์ได้รับการตรวจสภาพอีกครั้งหลังจากลบรอยเสร็จแล้ว

**Acceptance Criteria:**
