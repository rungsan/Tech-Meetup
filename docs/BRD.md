## 1. Executive Summary / วัตถุประสงค์

ระบบตรวจสภาพรถยนต์ (Car Inspection System) ของ ABC ประกันภัย พัฒนาและออกแบบขึ้นเพื่อ **อำนวยความสะดวกและเพิ่มประสิทธิภาพการทำงาน** ของเจ้าหน้าที่ภายในบริษัท ตลอดจนการติดตามกระบวนการตรวจสภาพรถยนต์ผ่าน Surveyor ที่ได้รับมอบหมาย

ระบบประกอบด้วย 2 ส่วนหลัก:

1. **ระบบตรวจสภาพรถยนต์ (Car Inspection System)** — สำหรับเจ้าหน้าที่ภายใน ABC ประกันภัย เพื่อสร้าง จัดการ แก้ไข และติดตามสถานะการตรวจสภาพรถยนต์
2. **ระบบ E-Motor Survey** — สำหรับเจ้าหน้าที่ของบริษัท Surveyor (ผู้ตรวจสภาพรถภายนอก) เพื่อรับงาน ดำเนินการตรวจ และส่งผลการตรวจสภาพกลับให้ ABC ประกันภัย

## 2. Business Context / ปัญหาและความต้องการทางธุรกิจ

### 2.1 ปัญหาเดิม

- กระบวนการตรวจสภาพรถยนต์ก่อนทำประกันขาดระบบรองรับแบบครบวงจร
- การติดตามสถานะงานและนัดหมายลูกค้าทำผ่านช่องทางที่กระจัดกระจาย ขาดประสิทธิภาพ
- การประสานงานระหว่างเจ้าหน้าที่ ABC และบริษัท Surveyor ภายนอกขาดระบบกลาง
- การรายงานผล KPI และสรุปค่าใช้จ่าย (ตั้งเบิก) ต้องทำด้วยตนเองและใช้เวลานาน
- ขาดการติดตามแบบ Real-time และ Dashboard สำหรับผู้บริหาร

### 2.2 ความต้องการทางธุรกิจ

- มีระบบกลางสำหรับบริหารจัดการงานตรวจสภาพรถยนต์ครบวงจร ตั้งแต่รับแจ้งงานจนถึงปิดงาน
- รองรับการทำงานแบบ Fleet (ตรวจสภาพรถหลายคันในคราวเดียว) สำหรับลูกค้าองค์กร
- มีระบบแจ้งเตือนอัตโนมัติผ่านอีเมล เมื่อมีการรับงาน ปฏิเสธงาน หรือส่งผลตรวจสภาพ
- บริษัท Surveyor สามารถรับงาน อัปโหลดรูปผลตรวจสภาพ และส่งผลกลับทางออนไลน์ได้
- สามารถออกรายงานหลากหลายรูปแบบ (สถานะงาน, สรุปผล, KPI, การตั้งเบิก) พร้อม Export Excel/PDF
- มีระบบควบคุมสิทธิการเข้าใช้งาน (Role-based Access Control) ที่ยืดหยุ่น

## 5. Scope

### 5.1 In Scope (สิ่งที่อยู่ในขอบเขต)

**ระบบ Car Inspection System:**
- หน้าจอ Login ผ่าน Microsoft Azure AD
- Dashboard แสดงสรุปสถานะงานและกราฟ
- หน้าจอแจ้งตรวจสภาพรถใหม่ (รายคัน + Fleet สูงสุด 10 คัน)
- หน้าจอรายการของฉัน (My Work List)
- หน้าจอรายการตรวจสภาพ (All Inspection List)
- รายงานสถานะข้อมูลการตรวจสภาพรถยนต์
- รายงานสรุปผลการตรวจสภาพ
- รายงาน KPI
- รายงานสรุปจำนวนการเข้าบันทึกงานตามช่วงเวลาของพนักงาน
- รายการตั้งเบิก (สรุปการตั้งเบิก + ตรวจสอบ/อนุมัติตั้งเบิก)
- การจัดการอีเมล Agent
- แก้ไขอีเมลจากเลข Job
- ลบรายงานผลตรวจสภาพ
- จัดการ Survey (บริษัท Surveyor)
- แก้ไข Agent/Source/รายละเอียดรถ
- การจัดการข้อมูลยี่ห้อ รุ่นรถ
- Role Setting

**ระบบ E-Motor Survey:**
- หน้าจอ Login ด้วย Username/Password
- รายการตรวจสภาพรถยนต์ (รับงาน/ปฏิเสธงาน/อัปโหลดผล/ส่งผล)
- สรุปรายการตั้งเบิก (รายเดือน/รายปี)
- การจัดการผู้ใช้งาน
- Role Setting

### 5.2 Out of Scope (สิ่งที่ไม่อยู่ในขอบเขต)

- ระบบรับประกันภัยรถยนต์ (Insurance Underwriting System) หลัก
- การชำระเงินออนไลน์
- แอปพลิเคชันมือถือ (Mobile Application)
- ระบบจัดการเอกสารประกันภัย (Policy Management)
- การบูรณาการกับระบบภายนอกอื่นนอกเหนือจากที่ระบุใน BR-003

## 6. System Modules / Features Overview

### 6.1 ระบบ Car Inspection System

#### Module 1: Authentication & Authorization
- Login ผ่าน Microsoft Azure AD
- Role-based Access Control ผ่าน ADD Group
- กำหนดสิทธิการเข้าถึงเมนูตาม Role (Car Ins Staff, UW, Admin)

#### Module 2: Dashboard
- แสดงจำนวนงานแต่ละสถานะ (8 สถานะ) พร้อม Clickable Card
- กราฟแท่งเปรียบเทียบปริมาณการตรวจสภาพรถ 3 ปีย้อนหลัง
- กราฟค่าใช้จ่าย 3 ปีย้อนหลัง
- Top 5 Survey (Pie Chart)
- สถานะงานทั้งหมด (Bar Chart)
- กรองข้อมูลตามช่วงเวลา (กำหนดเอง, 30/60/90 วันย้อนหลัง)

#### Module 3: แจ้งตรวจสภาพรถใหม่
- สร้างรายการตรวจสภาพแบบรายคัน (บุคคลทั่วไป / นิติบุคคล)
- สร้างรายการแบบ Fleet (สูงสุด 10 คันต่อรายการ)
- รองรับรถป้ายแดง
- ระบุ Source, Agent, ฝ่ายธุรกิจ (รองรับ 30+ สาขาทั่วประเทศ)
- ระบุยี่ห้อ/รุ่นรถ (Non EV / EV)
- ระบุสถานะการนัด และหมายเหตุ

#### Module 4: รายการของฉัน (My Work)
- แสดงรายการงานที่เจ้าหน้าที่รับผิดชอบ
- ดำเนินการตาม Workflow 5 ขั้นตอน:
  1. ติดตามนัดหมายลูกค้า
  2. ส่ง SV ออกตรวจสอบ
  3. ติดตาม SV
  4. รอผลตรวจรถยนต์ (ตรวจสอบรูป, Revise รูป)
  5. ตรวจรถยนต์แล้ว
- จัดการกรณีลบรอย Remark (2 รูปแบบ)
- ดู Job History
- ส่งข้อความ SMS

#### Module 5: รายการตรวจสภาพ (All Inspections)
- แสดงรายการตรวจสภาพทั้งหมดของทุกเจ้าหน้าที่
- ค้นหาและกรองข้อมูลตามหลายเงื่อนไข
- Assign งานให้ตนเอง

#### Module 6: รายงาน (Reports)
- **รายงานสถานะข้อมูลการตรวจสภาพรถยนต์:** ค้นหาและ Export ข้อมูลสถานะงานทั้งหมด
- **รายงานสรุปผลการตรวจสภาพ:** สรุปผลการตรวจสภาพในช่วงเวลาที่กำหนด
- **รายงาน KPI:** ติดตามประสิทธิภาพการทำงาน
- **รายงานสรุปจำนวนการเข้าบันทึกงานตามช่วงเวลาของพนักงาน:** วิเคราะห์ productivity
- **รายการตั้งเบิก:** จัดการการเบิกจ่ายค่าตรวจสภาพ พร้อมอนุมัติ/ไม่อนุมัติ

#### Module 7: ตั้งค่า (Settings)
- **การจัดการอีเมล:** จัดการอีเมลของ Agent
- **แก้ไขอีเมลจากเลข Job:** แก้ไขอีเมลเฉพาะ Job
- **ลบรายงานผลตรวจสภาพ:** ลบรายงานโดยเจ้าหน้าที่ที่มีสิทธิ
- **จัดการ Survey:** เพิ่ม/แก้ไขข้อมูลบริษัท Surveyor นำเข้าข้อมูล Master
- **แก้ไข Agent/Source/รายละเอียดรถ:** แก้ไขข้อมูลงาน
- **การจัดการข้อมูลยี่ห้อ รุ่นรถ:** เพิ่ม/แก้ไข/นำเข้า/Export ข้อมูลรถ
- **Role Setting:** กำหนดสิทธิการเข้าถึงของแต่ละ Role

### 6.2 ระบบ E-Motor Survey

#### Module 1: Authentication
- Login ด้วย Username/Password (ตรวจสอบจาก Database)

#### Module 2: รายการตรวจสภาพรถยนต์
- Dashboard แสดงจำนวนงานตามสถานะ (งานใหม่, รับงาน, ส่งงานคืน SV, SV ส่งรายงานผลแล้ว, งานสำเร็จ)
- ค้นหาและกรองรายการงาน
- รับงาน / ปฏิเสธงาน
- อัปโหลดรูปผลตรวจสภาพรถ (Browse File)
- Preview/Delete รูป
- ส่งผลตรวจสภาพ (Complete)
- แก้ไขรูปตรวจสภาพตาม Comment ของ ABC
- ส่งข้อความเพิ่มเติมให้ ABC
- Export ข้อมูล Excel

#### Module 3: สรุปรายการตั้งเบิก
- แสดงรายการตั้งเบิกแต่ละเดือน (สร้างอัตโนมัติ)
- แสดงรายละเอียดตั้งเบิกรายเดือน
- Export PDF/Excel

#### Module 4: การจัดการผู้ใช้งาน
- เพิ่ม/แก้ไข/ลบ User ของ Surveyor
- กำหนด Role และสถานะผู้ใช้
- ตรวจสอบ Password ตาม Policy (อย่างน้อย 8 ตัว, ผสมตัวอักษร/ตัวเลข/อักขระพิเศษ)

#### Module 5: Role Setting
- กำหนดสิทธิการเข้าถึงเมนูของแต่ละ Role ในระบบ E-Motor Survey

## 7. Integration Requirements

### 7.1 Microsoft Azure Active Directory (Azure AD)
- ระบบ Car Inspection ใช้ Azure AD สำหรับ Single Sign-On (SSO) ของ ABC Staff
- ตรวจสอบ Role จาก ADD Group — ผู้ใช้ต้องอยู่ใน 1 ADD Group เท่านั้น
- หากอยู่มากกว่า 1 Group จะไม่ได้รับสิทธิเข้าระบบ

### 7.2 API การรับรูปตรวจสภาพรถ
- รับรูปตรวจสภาพรถจาก External Source ผ่าน API ได้แก่:
  - สาขาส่งรูป
  - TQM ส่งรูป
  - ธุรกิจ 4 / SCB ส่งรูป
- ระบบจะแสดงแถบ "Car Inspect Image" เพื่อให้เจ้าหน้าที่ตรวจสอบรูปที่ API ส่งมา

### 7.3 Email System
- ส่งอีเมลอัตโนมัติ (Auto Email) เมื่อสร้างรายการตรวจสภาพรถใหม่
- ส่งอีเมลแจ้งเตือนไปที่ **Auto@ABC.co.th** เมื่อ Surveyor รับงานหรือปฏิเสธงาน
- รองรับการระบุอีเมลหลายที่อยู่ โดยใช้ ";" คั่น

### 7.4 Integration ระหว่าง Car Inspection และ E-Motor Survey
- เมื่อเจ้าหน้าที่ ABC กด "Complete" ในขั้นตอน "ส่ง SV ออกตรวจสอบ" ระบบจะส่งงานไปแสดงใน E-Motor Survey โดยอัตโนมัติ (สถานะ "งานเข้าใหม่")
- เมื่อ Surveyor กด "Complete" ส่งผลตรวจสภาพ ระบบจะอัปเดตสถานะใน Car Inspection

## 8. Non-Functional Requirements

### 8.1 Security (ความปลอดภัย)
- การ Authentication ผ่าน Microsoft Azure AD (ฝั่ง ABC)
- Password Policy สำหรับ E-Motor Survey:
  - ความยาวอย่างน้อย 8 ตัวอักษร
  - ประกอบด้วยอย่างน้อย 2 ใน 3 ประเภท: ตัวอักษร (a-z, A-Z), ตัวเลข (0-9), อักขระพิเศษ
- Role-based Access Control (RBAC) ควบคุมการเข้าถึงทุกเมนู
- การเปลี่ยน Role Setting มีผลต่อเมื่อ Login ใหม่ หรือเมื่อมีการเรียก API ตรวจสอบสิทธิ

### 8.2 Usability (ความง่ายในการใช้งาน)
- รองรับ Auto Complete ในฟิลด์ค้นหา Source, Agent, ยี่ห้อรถ, รุ่นรถ
- รองรับการ Pagination (10, 20, 50, 100 รายการต่อหน้า)
- มี Sort by สำหรับคอลัมน์สำคัญ

### 8.3 Data Management
- รองรับการ Export ข้อมูลในรูปแบบ Excel (.xlsx) สำหรับรายงานทุกประเภท
- รองรับการ Export รายการตั้งเบิกในรูปแบบ PDF
- รองรับการ Import ข้อมูล Master (ยี่ห้อ/รุ่นรถ, บริษัท Survey) ผ่าน Template ที่กำหนด

### 8.4 Real-time Notification
- แจ้งเตือน (Notification) เมื่อมีข้อความใหม่ใน Job History
- ส่งอีเมลอัตโนมัติสำหรับเหตุการณ์สำคัญ (สร้างงานใหม่, รับงาน, ปฏิเสธงาน)

### 8.5 Reporting
- รายงาน KPI และสรุปผลตรวจสภาพรองรับการแสดงข้อมูลตามช่วงเวลาที่กำหนด
- Dashboard แสดงข้อมูลเปรียบเทียบ 3 ปีย้อนหลัง (รวมปีปัจจุบัน)
- รายการตั้งเบิกสร้างอัตโนมัติทุกสิ้นเดือน ดูได้ตั้งแต่วันที่ 3 ของเดือนถัดไป

### 8.6 Compatibility
- ระบบเป็น Web Application รองรับการใช้งานผ่านเว็บเบราว์เซอร์มาตรฐาน
- E-Motor Survey ออกแบบสำหรับใช้งานภาคสนาม รองรับการอัปโหลดรูปภาพจากอุปกรณ์
